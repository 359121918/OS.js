Configuring and building

OS.js uses Grunt for building and configuring. Look at the configuration manual page for more information.

For a simplified guide on how to install OS.js look here.

For building other types of distributions (like Vagrant, Docker and NW), look at the repository INSALL file.

Build Directories

OS.js has two build directories:

dist

This is the "production" build. All files are concatenated (and optionally compressed) making it suitable for use in deployment.

dist-dev

This is the development build. It is not a "build" per-se, but rather symlinks to the source-code. Using this means you don't have to run any tasks to make your changes take effect (with some exceptions, like themes).

It's highly recommended you use this when making changes to OS.js because it has full debugging (logging to browser console) enabled.

Building

To build, simply run: 

grunt

Building spesific components

Look at the configuration manual page for more information about building spesific components.

Configuring

Via Grunt

See the configuration manual page for full instructions. 

# Basic examples

grunt config:get --name=client.Connection.RootURI

grunt config:set --name=client.Connection.RootURI --value=/custom/path

grunt config # rebuild config files

Manually

You can find all the configuration files in src/conf directory.

The files are loaded in alphabetical order (which is why they are prefixed with a number) and merged together into one final map.

It is highly encouraged that you write your own files instead of modifying the provided ones (because they may be affected by updates).

I provide an example in the repository.

If you make any changes, issue this command to rebuild configuration files (not needed if you run "grunt watch"): 

grunt config

Adding proxies

You can proxy requests through OS.js webserver natively by simply adding a configuration.

Create a file called (ex) src/conf/900-proxies.json and add 

{
  "server": {
    "proxies": {
      "/^\\/starts\\-with/": "http://remote-service/somewhere",
      "match-only/": "http://localhost:5000/here"
    }
  }
}

Then run: 

# For node server
grunt config

# For php server behind apache (note that this does not generate regex rules yet)
grunt generate:apache-htaccess
