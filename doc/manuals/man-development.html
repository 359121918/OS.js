Development Introduction

There's some manual pages that explains some aspects of the development process of OS.js:

  • Configuration system
  • Build system

Below you can find some information on how to get started with developing for OS.js:

Setting up a development environment

OS.js already comes with a development environment, using dist-dev.

To install all utilities, do: 

# Install all grunt tasks etc
$ npm install --development

# To automatically restart server when changes in codebase occur
$ npm install -g supervisor

# Unit testing support
$ sudo npm install -g mocha

Running the development server

Just use the launcher: 

$ ./bin/start-dev.sh

Style guide

You can read more about this on wiki.

You can check your code style for "problems" by using automated tools: 

# Check your javascript
grunt jshint jscs

# Check your css
grunt csslint

# Check your schemes (html)
grunt xml_validator

Code locations

src/client/javascript             # Client JavaScript
src/client/stylesheets            # Client Stylesheets
src/client/themes                 # Client theme resources
src/server/node                   # Server code
src/packages                      # Package sources (applications)
src/conf                          # Configuration files
src/build.js                      # Build system codebase
Gruntfile.js                      # Build system setup
bin                               # General scripts
doc                               # General documentation
dist                              # Production build output
dist-dev                          # Development environment

Using the build system

More information about this in the build manual.

Getting started with development

You can look at the tutorials, to get started creating your own apps and extensions.

For contributions, look at the document over at the Github repository.

Tips

Restart your application

You can reload and relaunch your application with: 

OSjs.API.relaunch('MyApplicationClassName')

TEST ALL THE THINGS!

To run all tests on the code, just use: 

grunt test
