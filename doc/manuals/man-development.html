Development Environments

For more information about configuration see the configuration section. If you run 
grunt watch
in the background it will automatically configure the environment when files are changed (or added/removed).

dist (production)

This is the production build of OS.js. All files have been compiled into single files (and can be compressed) to increase loading speed. This will however make it harder to debug.

To build simply do: 

grunt

# or:
grunt dist

dist-dev (development)

This is what you should use when developing or testing changes. It will load all files directly from source (with exception of config and theme files).

This command is a shortcut to perform all tasks that dist-dev requires: 

grunt dist-dev
