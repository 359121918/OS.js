Extension

For extending the client and server APIs. Can also be used as a plugin in the client (that will be loaded on startup).

Metadata

Key Description Required Default
enabled If your package is enabled or not No true
name The name of your package Yes  
description The description of your package No  
preload The list of resources to preload 
"preload":    [
  {"src": "main.js", "type": "javascript"},
  {"src": "main.css", "type": "stylesheet"},
  {"src": "foo.js", "type": "javascript", combine: false} // Do not combine this script in 'dist'
],
Yes  
copy List of files to copy into your 'dist' (useful for removing sensitive server-side stuff) 
"dist": {
  "copy": [
    "extension.js"
  ]
},
Yes  
conf List of config files that are applied just like src/conf. (Note: These are loaded with lowest priority) 
"conf": [
  "mycustomconfig.json"
]
Yes  
depends A list of dependencies you want to load. A dependency is a package name (classname). It only loads the sources, it will not run the application. 
"depends":    [
  "ApplicationSettings"
],
No  
uses If you set this, this extension will be loaded when the defined application is launched. This is useful if you want to have "plug-ins" 
"uses":    "ApplicationSettings"
No
