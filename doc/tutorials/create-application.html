Tutorial: Create your own application

Create from template

Simply run:

grunt create-package:default/MyName

The "default" is the name of the repository folder you want to place your package in.

You will end up with "main.js", "main.css" (for client), "api.js" (for server) and "metadata.json" (see below)

This will create some source files for you with usable code. You'll have a basic running application with a Window.

Update metadata

The metadata file holds general information about your application and stuff used in the building process.

You don't have to do this, but this shows what values you can put in

Key Description Required Default
className The name of your JavaScript namespace Yes  
enabled If your package is enabled or not No true
singular Allow only one instance of the application running No false
autostart If you want to start package on login No false
name The name of your package Yes  
description The description of your package No  
category The category of your package No other
icon The icon filename 
# From icon theme:
"icon": "category/name.png",

# From application directory
"icon": "./name.png",
Yes  
splash Show a splash screen when application starts No true
names Localization of the 'name' attribute 
"names" : {
  "no_NO" : "Norwegian name"
}
No  
descriptions Localization of the 'description' attribute 
"names" : {
  "no_NO" : "Norwegian description"
}
No true
mime What MIME types your package supports (Array with regexp) 
"mime": [
  "^text",
  "inode\\/x\\-empty",
  "application\\/x\\-empty",
  "application\\/x\\-lua",
  "application\\/x\\-python",
  "application\\/javascript",
  "application\\/json"
],
No  
compability Check for these browser features when starting application (Array of strings) 
# See OSjs.Utils.getCompability()

"compabiltiy": [
  "canvas", "blob", "audio", "video"
],
No  
preload The list of resources to preload 
"preload":    [
  {"src": "main.js", "type": "javascript"},
  {"src": "main.css", "type": "stylesheet"},
  {"src": "foo.js", "type": "javascript", combine: false} // Do not combine this script in 'dist'
],
Yes  

Install

Now run this command to add it to global manifest (make it visible in OS.js)

grunt manifest config packages:default/MyName

If you are using "dist-dev", you don't have to run the "packages" grunt command.

Tips

If you want to reaload your application when it has been updated simply run "OSjs.API.relaunch('ApplicationMyName')" in the developer console of your browser.
