Tutorial: Extend Server API

This tutorial show you how to add client (javscript) API extensions.

You can use this in combination with Server API Extensions

Method 1 - Core Extension

  1. Create a new file, ex: src/javascript/helpers/my-module.js
  2. Add the file to: src/javascript/helpers/my-module.js
  3. Rebuild with grunt core (dist), or grunt dist-dev-index (dist-dev)

Example file: 

(function(Utils, VFS, API) {

  /////////////////////////////////////////////////////////////////////////////
  // MODULE API
  /////////////////////////////////////////////////////////////////////////////

  // Just a basic pattern to keep methods
  var MyModule = {
    someMethod: function() {
    }
  };

  /////////////////////////////////////////////////////////////////////////////
  // EXPORTS
  /////////////////////////////////////////////////////////////////////////////

  // Available namespaces: API, VFS, Utils, Helpers, Core
  // Helpers is what I call the generic libraries... because they "help you"
  OSjs.Helpers.MyModule = MyModule;

})(OSjs.Utils, OSjs.VFS, OSjs.API);

Method 2 - Package Extension

Just create a new package:

  1. Create a new package with grunt create-package:MyModule
  2. Delete everything except package.json
  3. Change package.json to look like example below
  4. Add extension.js file as example below
  5. Rebuild with grunt packages config (dist), or grunt config (dist-dev)

package.json

{
  "className": "ExtensionMyModule",
  "type": "extension",
  "build": {
    "copy" : [
      "package.json",
      "extension.js"
    ]
  },
  "sources": [
    {"type": "javascript", "src": "extension.js"}
  ]
}

extension.js

Use the same code as in Method 1.
